# Contributing to Windows Penetration Toolkit

Thank you for your interest in contributing! We welcome improvements to the documentation, UI/UX, and simulation logic.

## ðŸš¨ Safety & Security Policy

**Strictly Prohibited Contributions:**
1.  **No Real Exploits:** Do not submit code that executes actual CVE exploits, shellcode, or malware.
2.  **No Remote Execution:** Do not submit code that connects to external targets or executes commands on remote systems.
3.  **No Obfuscation:** All code must be readable and transparent.

**Allowed Contributions:**
*   Frontend improvements (React/Tailwind).
*   New *simulated* scenarios (mocked JSON data).
*   Documentation updates.
*   Unit tests.

## Getting Started

1.  Fork the repository.
2.  Clone your fork: `git clone https://github.com/YOUR_USERNAME/windows-penetration-toolkit.git`
3.  Install dependencies: `npm install`
4.  Create a branch: `git checkout -b feature/amazing-feature`
5.  Make your changes.
6.  Push and open a Pull Request.

## Development

*   Run dev server: `npm run dev`
*   Run tests: `npm test`
*   Lint code: `npm run lint`
